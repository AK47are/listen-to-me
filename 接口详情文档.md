## 刷新令牌

POST `/common/auth/refresh` 刷新 Token

特别说明：请求已通过 JWT Filter 校验，Spring Security 上下文已包含 userId。返回时除新 Token 外，还返回用户最新信息，实现无感刷新

入参详情：无

出参详情：[LoginVO](实体类文档.md#LoginVO)

```mermaid
sequenceDiagram
    participant Client as 客户端
    participant Auth as 认证服务
    participant DB as 数据库

    Note over Client,Auth: 请求已通过 JWT Filter 校验，Spring Security 上下文已包含 userId

    Client->>Auth: POST /common/auth/refresh<br/>Header: Authorization: Bearer {current_jwt}
    Auth->>Auth: 从 Spring Security 上下文获取 userId
    Auth->>DB: 获取用户最新数据（昵称、头像、状态等）
    DB-->>Auth: 返回用户数据
    alt 用户已封禁
        Auth-->>Client: 403 Forbidden<br/>{code: "ACCOUNT_DISABLED", msg: "账号已被封禁"}
    else 用户正常
        Auth->>Auth: 生成新的 JWT
        Auth-->>Client: 200 OK<br/>LoginVO（包含最新用户数据）
        Client->>Client: 保存新 JWT 并更新本地用户信息
    end
```

## 获取播放签名

GET `/common/file/sign` 获取播放签名

特别说明：签名即临时 URL，同时负责 play_count 计算。为防止恶意刷量，同一用户对同一音频短时间内多次请求只计一次，用 Redis 记录，过期时间 1 小时

入参详情：`audioId`

出参详情：`String`（预签名URL）

```mermaid
sequenceDiagram
    participant Client as 客户端
    participant Auth as 认证服务
    participant DB as 数据库
    participant Minio as MinIO
    participant Redis as Redis

    Client->>Auth: GET /common/file/sign?audioId=1024
    alt audioId 为空
        Auth-->>Client: 400 Bad Request<br/>{msg: "音频ID不能为空"}
    else
        Auth->>Auth: 从 Spring Security 上下文获取 userId（可选）
        Auth->>DB: 查询音频信息（trialDuration, isPaid, 试听文件是否存在）
        alt 音频不存在
            Auth-->>Client: 404 Not Found<br/>{msg: "音频不存在"}
        else 存在
            alt 用户已登录
                Auth->>Redis: 检查是否已计数（key: user:play:counted:{userId}:{audioId}）
                alt 未计数
                    Auth->>DB: 音频 play_count +1
                    Auth->>Redis: 设置计数标记，过期时间 1 小时
                end
            end

            alt 用户已购买 或 音频免费
                Auth->>Minio: 生成完整音频文件的预签名URL
                Minio-->>Auth: 返回完整音频URL
                Auth-->>Client: 200 OK<br/>{完整音频URL}
            else 未购买且音频付费
                alt trialDuration > 0 且试听文件已生成
                    Auth->>Minio: 生成试听片段文件的预签名URL
                    Minio-->>Auth: 返回试听URL
                    Auth-->>Client: 200 OK<br/>{试听URL}
                else 试听未就绪或无试听
                    Auth-->>Client: 403 Forbidden<br/>{msg: "请购买后收听完整版"}
                end
            end
        end
    end
```
