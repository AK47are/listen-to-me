### 后端

#### 方法命名

取名规范为「动作」+「类名」+「限定」。以 `Audio` 类为例，规范如下：

| 业务动作 | Controller     | Service         | Mapper            |
| ---- | -------------- | --------------- | ----------------- |
| 新增   | `saveAudio`    | `addAudio`      | `insertAudio`     |
| 修改   | `updateAudio`  | `modifyAudio`   | `updateAudio`     |
| 删除   | `removeAudio`  | `deleteAudio`   | `deleteAudio`     |
| 详情   | `getAudio`     | `findAudioById` | `selectAudioById` |
| 分页   | `getAudioPage` | `findAudioPage` | `selectAudioPage` |
| 列表   | `getAudioList` | `findAudioList` | `selectAudioList` |

批量操作必须带 `Batch` 后缀。如 `deleteAudioBatch(List<Long> ids)`
分页查询：后缀加 `Page` 必须使用 `Query` 类（例如最简单的 `PageQuery`）

如果名字过长，可以适当简化。


#### 实体类

实体类取名主要要考虑到复合对象，简单类按功能命名如下：

- PO 类：名字与数据库表对应，Mybatis Plus 默认通过实体类名来找表名
- DTO 类：「主要操作类 + 业务 + 动作 + DTO」，例如 `UserLoginDTO`，比较灵活
- 条件查询类：用来封装前端查询参数，后缀带 `Query`
- VO 类：「主要操作类 + 业务 + VO」
	- 如果 VO 涉及了多个表的字段，应该使用基本 VO （即去除了敏感信息的 PO）进行组合

#### 日志

编写规范如下：`log.debug("当前情况 - (做了什么：可选): 关键数据 {}", data);`，例如：

- 登录时：`log.debug("用户登录 - 账号: {}", username);`
- 传文件：`log.debug("音频上传 - 文件名: {}, 大小: {}", fileName, size);`
- 查数据：`log.debug("查询详情 - ID: {}", id);`

> 务必在关键节点添加 `log`，少的时候可能没啥用，但是累计到一定数量，喂给 ai 能直接分析出不少问题。

#### 文件上传路径

规范为 `{业务模块}/{日期: 年月日}/{细分模块: 可选}/{UUID}_文件名.{后缀}`。

#### 工具类

工具类统一使用**静态方法**，不要作为容器调用，如果需要注入 `application.yaml` 里的字段，可以利用 [依赖注入（DI）](../../../03-课程学习/黑马%20Java%20Web%20教程/Spring%20Boot-基础.md#依赖注入（DI）) 里的「普通方法注入」来得到。例如下面 `JwtUtils` 的应用：

```java
public class JwtUtils {

    private static SecretKey KEY;
    private static Long EXPIRATION;

    @Value("${jwt.secret}")
    public void setSecret(String secret) {
        JwtUtils.KEY = Keys.hmacShaKeyFor(secret.getBytes(StandardCharsets.UTF_8));
    }

    @Value("${jwt.expiration}")
    public void setExpiration(Long expiration) {
        JwtUtils.EXPIRATION = expiration;
    }

	// 注意是 static
    public static String generateToken(String userId, String username) {
		......
    }

	......
}
```

> 使用「普通方法注入」是因为另外两种注入是通过「类实例」作为桥梁，普通方法注入可以让 Spring Boot 只用调用对应的 `setter` 方法。

#### Wrapper 编写

优先使用 MyBatis Plus 的 Wrappers 工具类来创建，支持链式反应，代码更简洁。

#### 异常

- 默认只使用 `BaseException`，通过 `message` 传递错误信息
- 只有需要自定义错误码时再自定义异常（例如 Token 错误等）
- 如果无需自定义，409 异常使用 `ConflictException`

#### 接口鉴权

如果接口设计只能让特定人使用，那么需要在 Controller 添加：

```
@PreAuthorize("hasAuthority('权限名（')")`
```

### 前端

#### 目录

```text
src
├── api                   // 01. 接口定义（与后端 Controller 一一对应）
│   ├── admin             // 管理端接口请求
│   ├── user               // 听众端接口请求
│   ├── creator           // 创作者端接口请求
│   └── common.js         // 公共接口（上传、登录）
├── assets                // 02. 静态资源
│   ├── icons             // SVG 图标
│   ├── images            // 图片
│   └── styles            // 全局 CSS / SCSS（变量、重置样式）
├── components            // 03. 公共组件（跨端复用）
│   ├── AudioPlayer       // 全局播放器组件
│   ├── FileUpload        // 统一上传组件
│   └── Layout            // 基础布局（侧边栏、顶栏）
├── hooks                 // 04. 组合式 API（封装通用逻辑，如：分页、倒计时）
├── router                // 05. 路由配置
│   ├── modules           // 路由拆分（admin.js, app.js）
│   └── index.js
├── store                 // 06. 状态管理 (Pinia)
│   ├── user.js           // 用户信息、Token
│   └── player.js         // 当前播放状态
├── utils                 // 07. 工具函数（request.js 封装、时间格式化）
└── views                 // 08. 页面视图（核心业务逻辑）
    ├── admin             // 管理后台页面
    ├── user               // 听众 H5/Web 页面
    ├── creator           // 创作者中心页面
    └── login             // 登录/注册公共页面
```
